# 予定共有アプリ「Asobi (アソビ)」 UI/UX設計書

---

## 1. 画面一覧（Screen Inventory）

| # | 画面名 | 目的 |
|---|--------|------|
| 1 | スプラッシュ | アプリ起動時のブランドロゴ表示 |
| 2 | オンボーディング (3枚) | アプリの価値を3ステップで説明 |
| 3 | ログイン/新規登録 | Apple ID / LINE / メールでの認証 |
| 4 | プロフィール設定 | 名前・アイコン・ステータス設定 |
| 5 | シフトパターン登録 | よく使うシフト（早番・遅番・夜勤等）の初期設定 |
| 6 | 友達招待 | LINE/QRコード/IDでの友達追加 |
| 7 | **ホーム** | 今日の予定サマリー + 直近の候補日提案カード |
| 8 | **カレンダー** | 月/週表示 + 友達のシフト重ね合わせ |
| 9 | **候補日提案** | 空き日のカード一覧（天気・文脈付き） |
| 10 | **シフト入力** | ワンタップでのシフト/予定登録 |
| 11 | **グループ管理** | グループ一覧・作成・メンバー管理 |
| 12 | グループ詳細 | 特定グループのカレンダー重ね合わせ |
| 13 | **日程確定** | 候補日の投票・確定・通知送信 |
| 14 | 通知一覧 | 招待・確定・リマインダーの通知履歴 |
| 15 | 友達一覧 | 友達リスト・オンライン状態表示 |
| 16 | 設定 | アカウント・通知・プライバシー設定 |
| 17 | 予定詳細 | 確定済み予定の詳細（場所・メモ・メンバー） |

## 2. ユーザーフロー

### (a) 初回登録フロー

```
[スプラッシュ]
     |
     v
[オンボーディング]
  "シフトを入れるだけ"
  "友達の空きが見える"
  "遊べる日を自動提案"
     |
     v
[ログイン/新規登録]
  ├─ Apple ID
  ├─ LINEログイン
  └─ メール登録
     |
     v
[プロフィール設定]
  名前・アイコン設定
     |
     v
[シフトパターン登録]  ← スキップ可能
  よく使うシフトを登録
  (早番 9-17 / 遅番 14-22 / 夜勤 22-6 等)
     |
     v
[友達招待]  ← スキップ可能
  ├─ LINEで招待
  ├─ QRコード表示
  └─ IDで検索
     |
     v
[グループ作成]  ← スキップ可能
  グループ名・メンバー選択
     |
     v
[ホーム画面] ★ メインループ開始
```

### (b) 日常利用フロー

```
[ホーム画面]
     |
     ├──→ [シフト入力] ワンタップでシフト登録
     |         |
     |         v
     |    カレンダーに反映
     |
     ├──→ [候補日提案カード] ← ホーム画面に表示
     |         |
     |         ├─ カードタップ → [候補日詳細]
     |         |                    |
     |         |                    ├─ 天気確認 (3時間ごと)
     |         |                    ├─ メンバーの空き確認
     |         |                    └─ 「この日にしよう！」
     |         |                         |
     |         |                         v
     |         |                    [日程確定画面]
     |         |                      全員に通知送信
     |         |
     |         └─ スワイプで次の候補へ
     |
     └──→ [カレンダー画面]
              友達のシフトを色分け確認
              空き日をタップ → [候補日提案]へ
```

### (c) 招待受信フロー

```
[プッシュ通知 / LINEメッセージ]
  「○○さんからグループ招待」
     |
     v
[アプリ起動]
     |
     ├─ 未登録 → [新規登録フロー(a)]
     |               |
     |               v
     |           自動でグループに参加
     |
     └─ 登録済み → [招待確認ダイアログ]
                      |
                      ├─ 「参加する」→ グループに参加
                      |                  |
                      |                  v
                      |            [スケジュール共有確認]
                      |              "このグループにシフトを共有しますか？"
                      |                  |
                      |                  ├─ 全共有
                      |                  ├─ 空き/埋まりのみ共有
                      |                  └─ 共有しない
                      |                       |
                      |                       v
                      |                  [グループ詳細画面]
                      |
                      └─ 「あとで」→ 通知一覧に保存
```

```
│    ☀️ 🌤️ ☁️ 🌧️ ❄️                          │
│                                                │
│ トーン:                                        │
│  - 「楽しい」「ワクワク」「カジュアル」        │
│  - 堅苦しさを排除、友達間のコミュニケーション  │
│    を想起させるデザイン                         │
│  - 競合(TimeTree)が「シンプル・実用的」に      │

│                                                │
└────────────────────────────────────────────────┘
```

### 競合との視覚的差別化

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  TimeTree        → 緑 + シンプル + カレンダー重視│
│  シフトボード    → 青 + 実用的 + シフト特化      │
│  本アプリ(Asobi) → 紫 + 楽しい + 提案重視       │
│                                                  │
│  差別化ポイント:                                  │
│                                                  │
│  1. カード型UI:                                  │
│     カレンダーが主ではなく「候補日カード」が主   │
│     → Tinder的なスワイプ体験                     │
│                                                  │
│  2. 文脈付きイラスト:                            │
│     単なる「空き日」ではなく                     │
│     「飲みに行ける日☀️🍺」という提案            │
│                                                  │
│  3. 天気のビジュアル統合:                        │
│     候補日カードに天気を大きく表示               │
│     → 意思決定の重要要素を一目で                 │
│                                                  │
│  4. マッチ度バー:                                │
│     メンバーの参加率をプログレスバーで視覚化     │
│     → ゲーミフィケーション要素                   │
│                                                  │
│  5. ダークモード対応:                            │
│     紫ベースはダークモードとの相性が良い         │
│     背景 #1A1A2E + カード #16213E               │
│                                                  │
└──────────────────────────────────────────────────┘
```

```
┌──────────────────────────────────────────────────┐
```

### アニメーション・トランジション方針

```
┌──────────────────────────────────────────────────┐
│ 基本方針                                         │
│                                                  │
│  - iOS Human Interface Guidelines に準拠         │
│  - 軽快さを重視 (duration: 0.25-0.35s)          │
│  - spring animation を多用 (弾むような動き)     │
│  - 意味のあるアニメーションのみ (装飾的NG)      │
│                                                  │
│ 画面遷移                                         │
│                                                  │
│  タブ切り替え:                                   │
│    → クロスフェード (0.2s)                       │
│    → タブアイコンにバウンスアニメーション        │
│                                                  │
│  モーダル表示 (シフト入力等):                    │
│    → ボトムシート (UISheetPresentation)          │
│    → スプリングアニメーション (0.35s)            │
│    → 背景が暗くなるスクリム                      │
│                                                  │
│  カード詳細展開:                                 │
│    → ヒーローアニメーション                      │
│    → カードが元の位置から全画面に拡大 (0.3s)    │
│    → 共有要素トランジション (日付・天気)        │
│                                                  │
│ マイクロインタラクション                          │
│                                                  │
│  シフト登録完了:                                 │
│    → チェックマークのLottieアニメーション        │
│    → 軽いハプティックフィードバック (success)    │
│    → カレンダー上の該当日がパルスアニメーション  │
│                                                  │
│  候補日発見:                                     │
│    → ホーム画面のカードがスライドイン            │
│    → 星のパーティクルエフェクト (控えめ)        │
│    → バッジカウントが弾む                        │
│                                                  │
│  日程確定:                                       │
│    → 紙吹雪アニメーション (0.8s)                │
│    → 全員のアイコンが集まるアニメーション        │
│    → ハプティック (notification success)         │
│                                                  │
│  プルトゥリフレッシュ:                           │
│    → カスタムインジケーター (アプリロゴ回転)    │
│    → データ更新後にコンテンツがフェードイン      │
│                                                  │
└──────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────┐
│ パフォーマンス指針                                │
│                                                  │
│  - 60fps を常に維持                              │
│  - メインスレッドでの重い処理を避ける            │
│  - アニメーション中のユーザー操作をブロックしない│
│  - Reduce Motion 設定時はフェードに置き換える    │
│  - Lottie は軽量なもの (50KB以下) のみ使用     │
│                                                  │
└──────────────────────────────────────────────────┘
```
